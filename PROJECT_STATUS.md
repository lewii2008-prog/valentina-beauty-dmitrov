- QR-код ведёт на публичный сайт;
- хранится как ассет проекта;
- не размещается на странице без отдельного решения;
- фактический путь к QR-коду необходимо зафиксировать после отдельной проверки его расположения внутри _deploy.

## 9. Порядок безопасного обновления

1. Проверить git status.
2. Убедиться, что работа выполняется в _deploy на ветке main.
3. Создать отдельный резервный коммит или тег перед существенными изменениями.
4. Внести только согласованные изменения.
5. Проверить локальную версию.
6. Проверить desktop и mobile.
7. Проверить отсутствие ошибок JavaScript.
8. Проверить защищённые файлы.
9. Создать коммит.
10. Выполнить push в main только после согласования.
11. Дождаться публикации GitHub Pages.
12. Проверить публичную страницу.
13. При проблемах использовать стабильный коммит или тег.

## 10. Проверка перед публикацией

- русский язык страницы;
- кодировка UTF-8;
- title и description;
- меню и якорные ссылки;
- подписи формы;
- мобильное меню;
- услуги;
- цены;
- FAQ;
- портфолио;
- сертификаты;
- кнопки и ссылки;
- изображения;
- отсутствие горизонтальной прокрутки;
- отсутствие ошибок JavaScript;
- prefers-reduced-motion;
- robots.txt;
- sitemap.xml;
- файлы подтверждения Google и Яндекс;
- .nojekyll.

## 11. История ключевых решений

- выбран GitHub Pages;
- сайт публикуется из отдельного репозитория _deploy;
- публикационная ветка — main;

- sitemap.xml отправлен;
- Яндекс Вебмастер ограничивает добавление подпапки GitHub Pages;

- анимации должны быть лёгкими, премиальными и без внешних библиотек;
